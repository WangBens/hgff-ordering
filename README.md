# 🚀 凰貴妃茶堂 - LINE 線上點餐系統開發全紀錄

> **開發日誌與心得**
> 這是一段從無到有、充滿挑戰的開發旅程。在專案初期，我對整體架構與功能流程並不清楚。透過釐清需求，我將系統拆解為「菜單顯示」、「加入購物車」、「訂單管理」與「結帳計算」等核心模組。在實作過程中，經歷了資料格式不一、按鈕失效、購物車無法正確累加、畫面未同步更新，甚至型別錯誤導致金額計算異常等各種問題。透過不斷地重構邏輯、分離功能與優化 UI，我最終完成了一套具備商業水準的系統。這段過程讓我學會如何獨立拆解問題、除錯與優化邏輯，大幅提升了程式開發的能力。

---

## 一、 系統架構與資料庫設計 (後端)

本系統確立以 Google Sheets 作為輕量級資料庫，搭配 Google Apps Script (GAS) 提供 API 服務，並完美整合 LINE LIFF 達成無縫登入的架構。

* **自動化建表：** 撰寫了 `ensureSheetsExist` 函式，讓系統初次運行時，自動建立包含系統設定、管理員名單、會員餘額、交易明細、菜單、客製化選項、訂單紀錄與促銷活動等資料表。
* **安全防護機制：** 實作了 LINE Token 驗證 (`verifyLineToken`)、API 請求的 Rate Limit（防止惡意連點），以及 Request ID 快取機制來避免重複提交訂單。
* **強大的折扣引擎：** 在後端開發了複雜的計價模型 (`calculateBackendTotal`)，支援「全館/系列/單品買 X 送 Y」以及「指定組合套餐價」。系統會將購物車攤平 (Flatten) 並由高價至低價優先折抵，確保對消費者最有利。

---

## 二、 顧客端介面開發 (INDEX.html)

為了讓顧客有最好的手機點餐體驗，透過 LINE LIFF 打造了專屬的顧客端網頁。

* **會員與錢包系統：** 顧客開啟網頁即自動帶入 LINE 資訊，支援一鍵加入會員。系統能即時顯示會員專屬餘額，並提供完整的歷史交易與訂單明細查詢。
* **動態菜單與跑馬燈：** 讀取後台設定，自動產生分類菜單。若有啟用的促銷活動，頂部會自動顯示活動跑馬燈，並獨立拉出「今日優惠組合」專區。
* **高度客製化購物車：** 顧客可以自由選擇杯型 (L/XL)、甜度、冰塊，並且支援「複選加料」。
* **無縫結帳體驗：** 支援「現場結帳」與「餘額扣款」兩種模式。結帳畫面上能即時、精準地顯示系統自動套用的活動折抵明細與最終總額。

---

## 三、 老闆管理端開發 (BOSS.html)

後台系統是營運的核心，設計了直覺的圖形化介面供管理員操作。

* **門市狀態與即時看板：** 可一鍵切換「營業中/休息中」狀態。訂單看板每 15 秒自動刷新，提供「接單、完成、已領」的訂單流程狀態按鈕。
* **菜單與活動管理：** 支援快速上架商品與雙杯型價格設定。強大的活動設定介面可針對單品或組合設定優惠，並支援隨時下架商品或停用活動（系統具備防呆機制，避免誤下架綁定活動中的商品）。
* **數位錢包管理：** 老闆可以直接在後台為指定顧客手動執行「儲值」或「扣款」，並填寫帳務備註，所有資金流動皆會寫入交易明細中供稽核。

## 四、 關鍵技術挑戰與完美解法

在系統升級為「商業級」穩定度的過程中，遭遇並克服了以下核心技術問題：

**⚠️ 挑戰 1：後台訂單數量的「假顯示」問題**
* **狀況：** 因為後端計價為了精準計算折扣，將所有同款飲料拆解成「單杯」陣列。這導致老闆看後台時，買 5 杯紅茶會變成 5 行獨立的紀錄，不易閱讀。
* **解法：** 在前端 `BOSS.html` 的 `renderOrders` 中導入 `groupedItems` 邏輯。將擁有相同「杯型、品名、甜度、冰塊、加料」的品項重新聚合，正確加總 `qty`，讓明細合併顯示為「[L] 紅茶 x5」，大幅提升後台閱讀與備餐效率。

**⚠️ 挑戰 2：前後端優惠計算不一致的致命傷**
* **狀況：** 顧客在前端購物車看到的金額，與送出訂單後端實際扣款的金額出現落差。前端原本的計算邏輯不夠嚴謹，無法完美處理複雜的「組合價」與「買送」交錯折扣。
* **解法：** 執行邏輯百分之百同步。將後端嚴謹的攤平排序計價邏輯 (`calculateBackendTotal`)，完整且無誤差地移植到前端的 `calculateCart()` 中。確保顧客在購物車看到的折扣金額，與最終結帳扣款金額絕對一致。

**⚠️ 挑戰 3：API 延遲導致畫面卡死 (UX 痛點)**
* **狀況：** 當使用者網路不穩或 Google 伺服器回應較慢時，前端使用 `fetch()` 呼叫 GAS 會陷入無限等待，畫面一直卡在「處理中...」，讓使用者以為系統當機。
* **解法：** 在前後端的 `callGAS()` 封裝中，全面導入 `AbortController` 實作 15 秒 Timeout (超時) 機制。若連線過久，系統會自動中斷請求，並溫柔地跳出「伺服器回應逾時，請稍後再試」的提示，完善防呆體驗。

**⚠️ 挑戰 4：殘留程式碼造成的語法錯誤**
* **狀況：** 在反覆修正前端計價邏輯時，不小心將一段迴圈遺漏在主函數外部，導致引發 `SyntaxError`，造成整個網頁崩潰白畫面。
* **解法：** 重新梳理程式碼區塊，精準定位並清除殘留的孤立程式碼，重構作用域 (Scope)，讓 `INDEX.html` 恢復正常渲染與運作。

## 五、 系統建置與部署教學 (Step-by-Step)

若要在新環境架設本系統，請依序執行以下五個階段：

**第一步：申請 LINE Developers 與 LIFF ID (準備前端入口)**
1. 前往 **[LINE Developers Console](https://developers.line.biz/)** 並登入你的 LINE 帳號。
2. 建立一個 **Provider** (供應商，例如：凰貴妃茶堂)。
3. 在該 Provider 下，建立一個 **LINE Login** 頻道的 Channel。
4. 進入該 Channel 的設定頁面，記下 **Channel ID** (此為後端所需的 `LINE_CLIENT_ID`)。
5. 切換到 **LIFF 頁籤**，點擊「Add」新增兩個 LIFF App。
6. 設定 **顧客端 LIFF**：Size 設為 `Full` 或 `Tall`，Scope 必須勾選 `profile` 與 `openid`。
7. 設定 **老闆端 LIFF**：Size 設為 `Full`，Scope 必須勾選 `profile` 與 `openid`。
8. 建立完成後，將這兩個 LIFF App 產生的 **LIFF ID** 分別記錄下來待用 (Endpoint URL 稍後填寫)。

**第二步：建立 Google Sheets 與 Apps Script (建置資料庫與後端 API)**
1. 開啟 Google 雲端硬碟，建立一個全新的 **Google 試算表 (Google Sheets)**。
2. 點擊試算表上方選單的 **擴充功能 > Apps Script**。
3. 清空編輯器內的預設程式碼，將 `GS` (後端程式碼) 完整貼上。
4. 找到程式碼最上方的 `const LINE_CLIENT_ID`，將引號內的數字替換為第一步取得的 **Channel ID**。
5. 點擊右上角的 **部署 > 新增部署作業**。
6. 類型選擇 **網頁應用程式 (Web App)**，執行身分設為 **我**，誰可以存取設為 **所有人 (Anyone)**。
7. 點擊「部署」並授權存取權限。
8. 部署完成後，複製畫面上顯示的 **網頁應用程式網址 (Web App URL)**，此為專屬的 `GAS_API_URL`。

**第三步：綁定金鑰與部署前端網頁 (INDEX.html 與 BOSS.html)**
1. 開啟程式碼編輯器，編輯 **INDEX.html (顧客端)**，將 `CUSTOMER_LIFF_ID` 與 `GAS_API_URL` 替換為前面步驟取得的值。
2. 編輯 **BOSS.html (老闆端)**，將 `BOSS_LIFF_ID` 與 `GAS_API_URL` 替換為前面步驟取得的值。
3. 將這兩個 HTML 檔案上傳至支援 HTTPS 的靜態網頁代管空間 (例如：GitHub Pages、Vercel、Netlify 等) 並發布。
4. 取得這兩個檔案發布後的 **公開網址** (例如：`https://yourdomain.com/INDEX.html`)。

**第四步：完成 LIFF 綁定與系統開機初始化**
1. 回到 LINE Developers Console 的 **LIFF 頁籤**。
2. 編輯稍早建立的兩個 LIFF App，將第三步取得的前端公開網址，分別對應填入 **Endpoint URL** 欄位中並儲存。
3. **系統自動建表：** 使用手機或電腦的 LINE 開啟「老闆端 LIFF」連結（格式如：`https://liff.line.me/你的老闆端LIFF-ID`）。後端程式偵測到首次啟動，會自動在 Google 試算表中建立所需的所有資料工作表。
4. **設定管理員權限：** 打開 Google 試算表，切換到「管理員名單」工作表，在「LINE UserId」欄位填入你自己的 LINE User ID (可從「會員與餘額」表單查看)。
5. **正式營運：** 重新開啟老闆端 LIFF，此時已具備管理員權限。接著即可在後台進行「上架菜單」、「設定促銷活動」、「開啟門市點餐」等營運操作，顧客端也同步正式上線！

**第五步：設定 LINE 官方帳號圖文選單 (打造專屬點餐入口)**
系統建置完成後，最後一步是讓顧客能方便地在 LINE 聊天室中打開點餐系統。
1. 準備一個 **LINE 官方帳號 (LINE Official Account)**。
2. 前往 **[LINE 官方帳號管理後台](https://manager.line.biz/)** 並登入。
3. 選擇你的店鋪官方帳號進入後台。
4. 在左側選單找到 **「主頁」 > 「圖文選單」**，點擊右上角的「建立」。
5. **基本設定：** 輸入標題與顯示期間（可設為無限期）。
6. **版面設定：** 上傳設計好的點餐按鈕圖片（例如：版型選擇單一滿版大按鈕）。
7. **動作設定：** 將類型選擇為 **「連結 (URL)」**。
8. **填入網址：** 將你在「第一步」取得的 **顧客端 LIFF URL**（格式如：`https://liff.line.me/你的顧客端LIFF-ID`）填入連結欄位中。
9. 點擊「儲存」，顧客進入官方帳號聊天室時，下方就會出現精美的點餐按鈕，點擊即可無縫開啟點餐系統！
